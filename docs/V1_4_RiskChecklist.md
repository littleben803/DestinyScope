# DestinyScope V1.4 Risk Checklist

更新日期：2026-05-29

## 1. 技术风险

- llama.cpp framework 在不同构建环境中可能链接失败。
- Debug / TestFlight / Release 的条件编译边界可能出错。
- 本地模型入口可能误入 Release。
- `TextRefinerFactory.makeDefaultRefiner()` 可能被误改。
- 异步生成可能造成 UI 状态错乱。

缓解：

- 小步实现。
- 每阶段运行 Debug / Release 构建。
- 明确 Release 不展示实验入口。
- 保持默认 refiner 不变。

## 2. 设备性能风险

- iPhone 17 Pro Max 约 1 秒级，但不代表所有设备。
- iPhone 12 mini 约 4 到 5 秒，体验偏慢。
- 连续生成可能发热或耗电。
- 内存峰值尚未通过 Instruments 完整记录。

缓解：

- 设备 tier 检测。
- Tier C 禁用。
- 超时回退。
- 继续补充 Instruments 数据。

## 3. 低端设备风险

- A14 / 小内存设备可能等待过长。
- 老设备可能加载失败或卡顿。
- 未识别设备如果默认启用会扩大风险。

缓解：

- 未识别设备默认禁用。
- Tier C 不展示入口。
- 保持模板路径为默认体验。

## 4. 隐私风险

- 误记录完整模型输入输出。
- 误上传日志。
- 用户误以为模型输入会上传。
- 未来模型下载会引入网络说明复杂度。

缓解：

- V1.4 不上传日志。
- 不记录完整润色文本。
- 仅本地记录必要运行状态。
- 隐私说明明确本地处理。

## 5. 审核风险

- 算命类 App 可能被认为夸大或误导。
- AI / 模型能力可能被误解为预测能力。
- 截图或元数据若写“AI 算命”会增加风险。

缓解：

- 不在正式元数据宣传 AI / 本地模型。
- 文案强调传统文化、自我探索、本地文本润色。
- 继续保留免责声明。

## 6. License / Notice 风险

- Qwen GGUF 实际来源可能未确认。
- llama.cpp / ggml notice 可能未随包展示。
- TestFlight 也是分发，不能忽略 license。

缓解：

- 人工确认 Qwen base / GGUF / llama.cpp license。
- 准备 App 内开源许可页面。
- 未完成前不随 App 分发模型。

## 7. 用户误解风险

- 用户可能认为模型生成了新的命理结论。
- 用户可能认为润色结果比原始结果更“准确”。
- 用户可能把润色预览当作最终判断。

缓解：

- 标注“润色预览”。
- 原始结果始终保留。
- 不覆盖原始模板文本。
- 说明不生成新的命理结论。

## 8. 模型输出安全风险

- 输出可能包含绝对化表达。
- 输出可能泄露特殊 token。
- 输出可能重复免责声明。
- 输出可能编造四柱、五行、十神、大运、流年。

缓解：

- SafetyChecker。
- 特殊 token 清理。
- 输出长度限制。
- 安全失败回退。

## 9. 包体 / 分发风险

- 当前模型约 `491.4 MB`。
- 内置模型会显著增大包体。
- 模型下载需要 CDN、校验、断点续传和隐私说明。

缓解：

- V1.4 不做正式模型下载。
- TestFlight 阶段优先手动导入或开发者准备。
- 不提交模型文件。

## 10. 回退策略风险

- 回退不彻底可能让模型错误输出进入页面。
- 超时处理不当可能阻塞 UI。
- 连续失败可能造成重复点击和状态错乱。

缓解：

- 所有失败回退 `TemplateTextRefiner`。
- 不覆盖原始结果。
- UI 显示温和错误。
- 连续失败后禁用本次会话实验入口。
