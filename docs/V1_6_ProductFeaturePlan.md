# DestinyScope V1.6 Product Feature Plan

## 1. V1.6 定位

V1.6 是核心产品功能打磨版本。

重点：

- 不考虑上架。
- 不考虑 TestFlight 上传。
- 不扩大本地模型生产化范围。
- 不默认启用本地模型。
- 不新增高风险命理能力。
- 继续保持完全 Native、本地优先、隐私友好。
- 重点提升首次使用体验、复用能力、结果分享、知识库留存和历史回看。

V1.6 不以“增加命理预测能力”为目标，而是让已有本地命理引擎、模板解读、知识库和历史记录更容易理解、复用和管理。

## 2. V1.6 核心目标

### P0 必须做

- 首次使用说明 / Onboarding。
- 常用出生资料 / 本地 Profile 方案。
- 结果页复制与纯文本分享文案。
- 知识库收藏和最近阅读方案。
- 历史记录收藏 / 置顶 / 快速复查方案。
- 首页信息架构优化。
- 所有新增数据仍只本地保存。
- 不引入账号、同步、服务端或网络。

### P1 可以做

- 结果页展开 / 折叠状态优化。
- 知识库推荐入门路径。
- 从结果页跳转相关知识文章。
- 本地数据管理页。
- 清空历史 / 清空收藏 / 清空 Profile 的统一数据管理入口。
- 分享文案模板多版本。

### 明确不做

- 不上传 TestFlight。
- 不准备 App Store 上架。
- 不做模型下载。
- 不做云端 AI。
- 不做 RAG。
- 不做流式 UI。
- 不做开放聊天。
- 不做付费订阅。
- 不做真人咨询。
- 不做改命、化解、避灾、转运类功能。
- 不做医疗、投资、婚恋确定性判断。
- 不把本地模型输出写入历史记录。
- 不让本地模型生成命理结论。
- 不做图片分享。
- 不做社区。

## 3. V1.6 与 V1.5 的关系

- V1.5 已完成体验和可读性打磨，包括结果页、知识库、历史记录和 Legal 体验。
- V1.6 在此基础上做产品功能增强，重点是用户复用、留存和本地数据管理。
- V1.6 仍然不改变默认主流程的安全边界。
- V1.6 保持 `makeDefaultRefiner()` 默认返回 `TemplateTextRefiner`。
- 本地模型实验保持受控、默认关闭、失败回退。
- V1.6 新增数据必须遵循本地保存、可删除、不上传、不同步原则。

## 4. V1.6 成功标准

- 新用户能理解 App 定位、隐私和使用边界。
- 用户能保存常用出生资料或快速复用输入。
- 用户能复制或分享结果摘要文本。
- 用户能收藏知识文章或查看最近阅读。
- 用户能更好地整理历史记录。
- 所有新增数据只保存在设备端。
- Debug / Release 构建稳定。
- 不引入新的隐私、审核或版权风险。
- 默认主流程仍不依赖本地模型。

## 5. 产品原则

- 先做本地体验闭环，再考虑分发或商业化。
- 先做轻量文本分享，不做图片分享。
- 先做本地 Profile，不做账号和同步。
- 先做收藏和最近阅读，不做社区。
- 先做数据管理和删除能力，再扩大本地数据保存范围。
- 所有新增文案继续使用“参考、建议、自我探索、传统文化学习”等表达。

## 6. 风险边界

V1.6 的主要风险不是算法准确性，而是本地数据和用户误解：

- 出生资料属于敏感个人信息，必须明确仅本地保存。
- 复制和分享文案不能被写成确定性预测。
- 收藏和最近阅读不能引入网络或账号假设。
- 历史记录增强不能重新计算或伪装成完整报告。
- 本地模型实验不能被包装成正式能力或默认能力。
