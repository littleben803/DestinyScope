# DestinyScope V1.1 本地小模型润色接口规划

更新日期：2026-05-27

## 1. 当前阶段边界

V1.1 阶段 6 只预留文本润色接口，不接真实模型：

- 不接在线 AI。
- 不接真实本地 LLM。
- 不接 Core ML、MLX 或 llama.cpp。
- 不加入模型文件。
- 不做模型下载。
- 不做流式输出。
- 不做 RAG 检索。
- 不改变现有命理计算和模板回答结果。

当前默认实现为 `TemplateTextRefiner`。它只返回本地模板文本，不请求网络，不调用模型，也不依赖 SwiftUI。

## 2. 接口职责

新增 `TextRefining` 抽象用于未来润色已有文本。输入是规则引擎和模板系统已经生成的内容，输出是可展示文案。

推荐职责边界：

- `LifeWeightEngine` 继续负责称骨计算。
- `LifeWeightInsightGenerator` 继续负责结构化洞察。
- `TemplateFortuneInterpreter` 继续负责五类命理师解读。
- `TemplateFortuneQuestionAnswerer` 继续负责本地命理问答结论。
- `TextRefining` 只负责表达润色，不能新增命理结论。

## 3. 输入输出结构

`TextRefiningInput`：

- `sourceText`：已经由本地规则或模板生成的原始文本。
- `purpose`：润色目的，如结果总评、问答回答、今日建议、知识解释或通用文本。
- `tone`：语气，如平和、简洁、古典、友好。
- `context`：可选上下文，只用于帮助润色，不用于生成新结论。
- `safetyRules`：安全边界，默认包含禁止绝对预测和现实决策建议等规则。

`TextRefiningOutput`：

- `text`：润色后文本。
- `wasRefined`：是否发生真实润色。
- `engine`：当前使用的实现名称，例如 `template`。
- `safetyNotice`：安全提示。

## 4. 默认实现

`TemplateTextRefiner` 是 V1.1 默认实现：

- 只 import Foundation。
- 不请求网络。
- 不使用 AI SDK。
- 不读取模型文件。
- `engine` 返回 `template`。
- 当前保持文本内容不变，只做空文本校验和统一输出封装。

`LocalLLMTextRefiner` 只是占位：

- 不导入 Core ML、MLX 或 llama.cpp。
- 不包含模型加载代码。
- 当前调用会抛出 `localModelNotAvailable`。
- TODO 留给后续第二阶段接入真实本地小模型。

`TextRefinerFactory` 当前默认返回 `TemplateTextRefiner`，不做设备检测、模型检测或模型文件读取。

## 5. 后续第二阶段接入方式

后续如果引入 0.5B 到 1.5B 本地小模型，建议按以下顺序推进：

1. 先保持 `TextRefining` 协议不变。
2. 新增真实本地模型实现，例如基于 llama.cpp、Core ML 或 MLX。
3. 只把模板系统生成的文本传入模型做润色。
4. 输出前继续执行安全规则检查。
5. 明确在隐私政策和审核备注中说明模型是否完全本地运行。

本地小模型不能负责命理计算、诗文匹配、权重判断、结论生成或知识真实性判断。

## 6. 安全规则

默认安全边界：

- 不做绝对预测。
- 不构成医疗、法律、财务、投资、婚恋或职业决策建议。
- 不写精准预测、改命、化解、避灾、必然发财、寿命疾病预测、投资确定性或婚姻确定性。
- 本地模型未来只能润色表达，不能生成命理计算结论。

如果未来模型输出与这些规则冲突，应丢弃模型输出并回退到模板文本。

## 7. 风险与待评估项

- 包体：本地模型会显著增加 App 体积。
- 内存：低端设备可能无法稳定加载 0.5B 到 1.5B 模型。
- 耗电：本地推理可能带来明显发热和耗电。
- 设备兼容：不同芯片和系统版本的性能差异需要真机测试。
- 审核文案：上线真实模型前，App Store 元数据不能提前宣传真实 AI 推理。
- 隐私说明：如果未来加入模型、RAG 或日志保存，需要同步更新隐私政策。
- 安全输出：模型可能改写出超出规则结论，需要保留模板回退和安全检查。
