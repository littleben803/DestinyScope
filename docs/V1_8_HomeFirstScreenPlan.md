# DestinyScope V1.8 Home First Screen Plan

## 1. 首页问题

当前首页经过 V1.6 信息架构优化后，包含：

- 品牌定位。
- 本地隐私提示。
- 历史草稿提示。
- 常用出生资料。
- 查询输入卡片。
- 最近历史摘要。
- 知识库入口提示。

V1.8 的新问题判断：

- 第一屏非查询信息仍偏多。
- 查询卡片不够聚焦。
- 小屏设备上查询按钮可能需要滑动才能看到。
- 常用资料、隐私说明、最近历史和知识库入口需要进一步后置或压缩。

## 2. 新首页第一屏目标

V1.8 首页第一屏必须优先服务生辰查询。

目标：

- 顶部保留简短品牌。
- 生辰查询卡片直接出现。
- 出生日期、出生时辰、查询按钮尽量首屏可见。
- 常用资料压缩展示，不喧宾夺主。
- 隐私提示短句内嵌。
- 最近历史后置。
- 知识库入口后置。
- 首页小屏仍可滚动，但查询路径不能被弱化。

推荐第一屏结构：

1. `DestinyScope` 短标题。
2. `东方命理 · 自我探索` 短副标题。
3. 生辰查询卡片。
4. 查询卡片内包含：
   - 出生日期。
   - 出生时辰。
   - 查询按钮。
   - 一句短隐私提示。
5. 常用资料入口压缩为横向短列表或折叠式入口。

## 3. 后置内容

以下内容应放在查询主路径之后：

- 最近历史摘要。
- 知识库入口。
- 较长的隐私说明。
- 本地数据管理提示。
- 产品定位长文案。

这些内容仍可保留，但不能影响第一屏查询路径。

## 4. 不做

V1.8 首页重构不做：

- 不移除隐私说明。
- 不隐藏结果边界。
- 不增加高风险营销文案。
- 不接在线功能。
- 不自动查询。
- 不上传出生信息。
- 不把本地模型能力包装为预测能力。

## 5. 验收标准

V1.8 阶段 3 实现时建议验收：

- iPhone 12 mini 小屏下查询按钮尽量首屏可见。
- iPhone 17 Pro Max 下第一屏以查询为中心。
- 出生日期和时辰选择清晰。
- 常用资料仍可复用，但不抢占主路径。
- 隐私提示仍可见。
- 最近历史和知识库入口后置。
- 不出现“精准预测”“改命”“化解”“避灾”等高风险表达。
