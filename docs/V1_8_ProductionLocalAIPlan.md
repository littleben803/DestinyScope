# DestinyScope V1.8 Production Local AI Plan

## 1. V1.8 定位

V1.8 是生产环境本地 AI 润色与首页主路径强化版本。

本阶段是一次明确的产品方向变更：

- 目标是快速形成可上线候选版本。
- 本地模型从 Debug / TestFlight 风格实验能力升级为生产候选能力。
- 首页第一屏必须聚焦生辰查询。
- 首页出生日期、出生时辰和查询按钮应尽量首屏可见。
- 本地模型文件计划直接打包进 App 安装包。
- 设备评分达到标准时，默认启用本地 AI 润色能力和入口。
- 模拟器默认启用本地 AI 能力，便于开发和验证。

V1.8 仍不做：

- 服务端。
- 在线 AI。
- 模型下载。
- RAG。
- 开放聊天。
- 流式 UI。
- 付费订阅。
- 上传用户数据。

## 2. 生产 AI 能力边界

本地模型只能做文本表达润色。

必须保持：

- 不生成新的命理结论。
- 不改变称骨计算。
- 不改变诗文匹配。
- 不改变 `LifeWeightInsight` 规则。
- 不改变命理问答结论。
- 不把本地模型润色结果写入历史记录。
- 不上传出生信息。
- 不上传命理结果。
- 不上传模型输入输出。
- 模型失败时回退模板文本。

本地模型可以处理的输入范围：

- 已有模板摘要。
- 已有行动建议。
- 已有安全提示。
- 经过结构化限制的非敏感上下文。

本地模型不能处理的输入范围：

- 用户完整出生资料明细。
- 历史记录列表。
- 常用出生资料 displayName。
- 知识库全文。
- 用户自由聊天输入。
- 未经过结构化处理的数据。

## 3. 默认启用策略

V1.8 默认启用策略：

- 设备评分 `>= 75` 时，默认开启本地 AI 润色。
- 模拟器默认开启。
- 用户不需要手动打开本地 AI 润色。
- 设备评分 `< 75` 时，默认使用模板文本。
- 低电量模式直接禁用本地 AI 润色。
- thermal serious / critical 直接禁用本地 AI 润色。
- 模型文件缺失直接回退模板。
- llama framework 缺失直接回退模板。
- 生成超时直接回退模板。
- 安全检查失败直接回退模板。

默认启用不代表模型参与命理计算。

本地模型只负责：

- 将已有模板文本润色为更自然的表达。
- 保留安全提示。
- 不新增结论。

## 4. 结果页展示策略

结果页必须保持模板结果始终可见。

推荐展示方式：

- 原始模板结果作为基准内容保留。
- 本地润色版作为“本地润色版”展示。
- 文案使用“表达润色，不改变命理结论”。
- 不写“AI 算命”。
- 不宣传精准预测。
- 不暗示模型比规则引擎更准确。
- 失败时温和回退，不打扰主流程。

建议文案：

- “本地润色版仅对已有模板文本做表达润色，不生成新的命理结论。”
- “如本地润色不可用，将自动使用模板文本。”
- “结果仅供娱乐、自我探索和传统文化学习参考。”

不建议文案：

- “AI 算命”。
- “精准预测”。
- “改命”。
- “化解”。
- “避灾”。
- “保证转运”。
- “必然发财”。

## 5. 首页策略

V1.8 首页第一屏目标：

- 第一屏聚焦出生日期、出生时辰和查询按钮。
- 查询按钮尽量首屏可见。
- Hero 压缩为短品牌和短副标题。
- 长隐私说明压缩为短句。
- 最近历史入口后置。
- 知识库入口后置。
- 常用资料保留，但不喧宾夺主。

推荐首页顺序：

1. 短品牌区：`DestinyScope` + `东方命理 · 自我探索`。
2. 生辰查询卡片：出生日期、出生时辰、查询按钮。
3. 常用资料轻量入口。
4. 本地隐私短提示。
5. 最近历史。
6. 知识库入口。

首页不做：

- 不移除隐私说明。
- 不隐藏结果边界。
- 不增加高风险营销文案。
- 不接在线功能。

## 6. 旧边界覆盖说明

用户在 V1.8 明确覆盖以下旧方向：

- 旧方向：本地模型只作为 Debug / TestFlight 实验路径。  
  V1.8 新方向：本地模型升级为生产候选能力。
- 旧方向：本地模型默认关闭。  
  V1.8 新方向：设备评分达标时默认启用本地 AI 润色。
- 旧方向：不把本地模型接入默认结果页。  
  V1.8 新方向：允许在受控边界下把本地润色版接入结果页展示。
- 旧方向：不提交或打包 GGUF 模型。  
  V1.8 新方向：允许规划将指定 GGUF 模型内置到 App Bundle，并通过 Git LFS 管理。
- 旧方向：首页继续产品信息层级打磨。  
  V1.8 新方向：首页第一屏必须优先服务生辰查询。

## 7. 仍保留的安全边界

V1.8 仍必须保留：

- 不接服务端。
- 不请求网络。
- 不接在线 AI。
- 不做模型下载。
- 不做 RAG。
- 不做开放聊天。
- 不做流式 UI。
- 不做付费订阅。
- 不上传用户数据。
- 不让模型生成命理结论。
- 不改变称骨计算、诗文匹配、洞察规则或问答结论。
- 安全检查失败必须回退模板。
- 低电量、过热、超时、设备不达标必须回退模板。
