# DestinyScope V1.3 TestFlight Experiment Plan

更新日期：2026-05-29

## 1. 阶段定位

本阶段只设计 TestFlight 内测开关，不直接实现。

边界：

- 不进入 App Store 正式版。
- 不默认开启。
- 不影响普通用户路径。
- 不改变当前结果页默认输出。
- 不替代 `TemplateTextRefiner`。
- 不生成新的命理结论。
- 不接服务端、不接在线 AI、不做模型下载。

本地模型实验只用于验证设备端文本润色体验，正式生产化需要后续单独决策。

## 2. 实验功能名称建议

推荐名称：

- 本地模型润色实验。
- 本地文本润色实验。

不建议名称：

- AI 算命。
- AI 大师。
- 精准预测。
- 改命助手。
- 转运分析。

命名原则：

- 强调“本地”和“润色”。
- 不暗示模型能预测未来。
- 不暗示模型能生成命理结论。
- 不把实验功能包装成核心算命能力。

## 3. 开关策略

### 默认状态

- 默认关闭。
- 用户手动开启。
- 首次开启必须展示说明。
- 用户关闭后应立即回退模板。
- App 重启后的状态需要在后续实现阶段单独设计，建议 TestFlight 初期默认每次安装或重装后关闭。

### 展示范围

| 构建类型 | 策略 |
| --- | --- |
| Debug | 可显示完整调试入口 |
| TestFlight | 可显示简化实验开关 |
| App Store Release | 默认不显示，除非后续单独决策 |

### 设备范围

| 设备等级 | 开关策略 |
| --- | --- |
| Tier A | 允许显示 TestFlight 实验开关 |
| Tier B | 可显示但必须提示“可能较慢或耗电”，也可暂不显示 |
| Tier C | 不显示，或显示为不可用 |

如果设备无法识别 tier，默认不显示或不可用。

## 4. 用户说明文案草案

首次开启前建议展示：

```text
本地模型润色实验

本功能使用设备端本地模型，对已经生成的模板文本做表达润色。它不会上传出生信息、命理结果、模型输入或模型输出，也不会生成新的命理结论。

本功能不会替代原有称骨计算、农历转换、诗文匹配和模板解释。部分设备上可能较慢、耗电或失败；失败时会自动回退本地模板文本。

该功能仅供 TestFlight 内测。所有内容仍仅供娱乐、自我探索和传统文化学习参考。
```

用户确认项建议：

- 我理解这是实验功能。
- 我理解结果仅供娱乐、自我探索和传统文化学习参考。

## 5. 开关影响范围

开启后只允许影响：

- `TextRefining` 润色。
- Debug / TestFlight 实验结果展示。
- 可选的“润色后文案预览”。

V1.3 推荐入口：

- 结果页底部“本地润色预览”卡片。
- 只在实验开关开启、设备支持且模型可用时展示。
- 用户手动点击“生成润色版”后才触发。
- 不自动生成。
- 不保存润色输出。
- 不覆盖原始模板文本。
- 不参与命理结论或后续问答推理。

不允许影响：

- 称骨计算。
- 农历转换。
- Poem 匹配。
- `LifeWeightResult`。
- `LifeWeightInsight` 结论生成。
- 命理问答的结论生成。
- 历史记录原始结果。
- 知识库内容。
- App Store Release 默认输出。

## 6. 回退策略

任意情况都回退 `TemplateTextRefiner`：

- 用户关闭开关。
- 设备不支持。
- 模型文件不存在。
- framework 不可用。
- 加载失败。
- 生成超时。
- 输出为空。
- 输出过长。
- 安全检查失败。
- 输出包含提示词泄露或特殊 token。
- 连续失败次数过多。
- 低电量模式。
- 过热状态。

回退时不应展示内部调试错误、模型路径或框架细节。用户只需要知道已使用本地模板文本。

## 7. 超时策略

初稿：

| 设备等级 | 超时策略 |
| --- | --- |
| Tier A | 3 秒 |
| Tier B | 5 秒 |
| Tier C | 不启用 |

超时后回退模板文本。

UI 不应长时间阻塞。后续实现阶段需要考虑异步执行、加载状态、取消和结果回填。

## 8. 实验数据记录策略

V1.3 暂不上传日志。

本地可以记录：

- 是否开启。
- 设备 tier。
- 是否成功。
- 是否 fallback。
- 耗时。
- 错误类型。

不记录：

- 用户出生信息。
- 完整命理结果。
- 完整模型输入输出。
- 可识别个人身份的信息。

如果未来要上传匿名性能数据，需要单独更新隐私政策、用户授权、审核备注和数据处理说明。

## 9. TestFlight 测试清单

必测：

- Tier A 设备开启 / 关闭开关。
- Tier B 设备开启 / 关闭开关。
- Tier C 设备不可用状态。
- 模型不存在时回退。
- framework 不可用时回退。
- 安全检查失败时回退。
- 生成超时时回退。
- 飞行模式下可用。
- 低电量模式行为。
- 深色 / 浅色模式。
- App 重启后开关状态是否符合预期。
- 连续多次生成后是否发热、卡顿或失败。
- 特殊 token 泄露是否被拦截。

## 10. 不建议现在做的事

- 不做 App Store 正式开关。
- 不做模型下载。
- 不做付费功能。
- 不做云端 AI。
- 不把模型输出接入默认结果页。
- 不宣传 AI 算命能力。
- 不把模型能力写入当前 App Store 生产元数据。

## 11. 阶段结论

V1.3 的 TestFlight 实验开关应默认关闭，只对符合设备分级策略的内测设备开放。开关只影响 `TextRefining` 润色实验，所有失败、超时、安全异常和用户关闭场景都必须回退 `TemplateTextRefiner`。
