# DestinyScope V1.2 本地小模型 PoC 计划

更新日期：2026-05-27

## 1. PoC 目标

V1.2 PoC 的目标是在不影响当前主流程的情况下，验证本地 0.5B 到 1.5B 小模型在 iPhone 端运行的可行性。

PoC 目标：

- 使用现有 `TextRefining` 接口接入实验实现。
- 输入为模板结果和结构化命理数据。
- 输出为润色后的文本。
- 验证模型加载、生成速度、内存峰值、耗电、发热和输出质量。
- 验证断网状态下是否可用。
- 验证安全边界是否可控。

模型不得决定命理结论。命理结论仍由 `LifeWeightEngine`、`TemplateFortuneInterpreter`、`LifeWeightInsightGenerator` 和 `TemplateFortuneQuestionAnswerer` 生成。

## 2. PoC 范围

PoC 只做实验验证：

- 只做 Debug / 实验入口。
- 不对普通用户默认开启。
- 不修改现有结果页默认输出。
- 不替换当前模板式解读。
- 不做模型下载。
- 不接服务端。
- 不接在线 AI。
- 模型文件先由开发者手动放入本地或 Bundle 测试。
- 不进入 App Store 生产版本。

不进入本阶段的内容：

- 不做生产级模型管理。
- 不做自动下载和增量更新。
- 不做付费订阅。
- 不做开放式聊天。
- 不做真实 RAG 检索。
- 不做模型输出直接上屏的生产路径。

## 3. PoC 输入设计

建议输入只使用已生成的结构化结果：

- `LifeWeightResult` 摘要：
  - 命格标题。
  - 农历生日展示文本。
  - 总重量文本。
  - 称骨诗文。
  - 年、月、日、时权重明细。
- `FortuneInterpretation`：
  - 总评。
  - 性格。
  - 事业。
  - 财运。
  - 关系。
  - 安全提示。
- `LifeWeightInsight`：
  - tags。
  - focusTitle。
  - focusDescription。
  - strengths。
  - cautions。
  - actionSuggestion。
  - safetyNotice。
- `FortuneAnswer`：
  - questionTitle。
  - intent。
  - answer。
  - safetyNotice。
- `rag_chunks` 中召回的少量文本：
  - PoC 初期可以手工传入 1 到 3 条 chunk。
  - 不做向量数据库。
  - 不做自动检索。
- `safetyRules`：
  - 禁止绝对预测。
  - 禁止新增命理结论。
  - 禁止医疗、法律、财务、投资、婚恋、职业决策建议。
  - 禁止改命、化解、避灾、必然发财、寿命疾病预测、投资确定性、婚姻确定性等内容。

## 4. PoC Prompt 设计原则

Prompt 必须明确：

- 你只能润色已有文本。
- 不能新增命理结论。
- 不能编造八字、四柱、五行、十神、大运、流年。
- 不能做绝对预测。
- 不能提供医疗、法律、投资、婚姻确定性建议。
- 输出要短，建议 200 到 500 字。
- 输出要保留免责声明。
- 如果输入信息不足，必须保持保守表达。
- 如果模型不确定，应回到“仅供娱乐、自我探索和传统文化学习参考”的表述。

Prompt 模板草案：

```text
你是 DestinyScope 的本地文本润色器。你只能润色用户已经看到的模板文本，不能新增任何命理结论。

任务：将下面的本地模板结果润色为更自然、温和、清晰的中文表达。

限制：
- 不要新增四柱、五行、十神、大运、流年等输入中没有的信息。
- 不要使用绝对预测。
- 不要提供医疗、法律、财务、投资、婚恋或职业决策建议。
- 不要写改命、化解、避灾、必然发财、寿命疾病预测、投资确定性或婚姻确定性。
- 输出 200 到 500 字。
- 最后保留安全提示。

结构化输入：
{{structured_context}}

待润色文本：
{{source_text}}
```

## 5. PoC 验收指标

功能指标：

- iPhone 真机可运行。
- 断网可用。
- 可通过 `TextRefining` 接口调用。
- 不影响当前默认模板输出。
- 单次回答 200 到 500 字。
- 输出包含安全提示。

性能指标：

- 首 token 时间可接受。
- 总生成时间可接受。
- 模型加载不导致 App 崩溃。
- 内存峰值可接受。
- 连续生成多次不明显泄漏内存。
- 设备发热和耗电可接受。

质量指标：

- 输出质量明显优于模板原文。
- 表达更自然、更连贯、更适合阅读。
- 不新增命理结论。
- 不编造八字、五行、流年等未提供信息。
- 安全文案不越界。

## 6. 不通过标准

出现任一情况，应暂停进入生产路径：

- 模型加载导致崩溃。
- 包体不可接受。
- 生成速度过慢。
- 首 token 时间明显不可接受。
- 内存峰值过高。
- 老设备体验不可接受。
- 输出经常胡编八字、五行、流年或其他未输入信息。
- 输出高风险建议。
- 输出包含绝对预测、改命、化解、避灾、必然发财等内容。
- 无法稳定保留免责声明。
- 模型 license 不允许 App 分发或商业使用。

## 7. PoC 结果记录建议

建议每次模型测试记录：

- 设备型号。
- iOS 版本。
- 模型名称和参数量。
- 量化格式。
- 模型文件大小。
- 首 token 时间。
- 总生成时间。
- 峰值内存。
- 是否断网可用。
- 是否发热明显。
- 输出样例。
- 是否出现安全越界。
- 是否建议继续评估。
