# DestinyScope V1.3 Device Tier Plan

更新日期：2026-05-29

## 1. 设备分级目标

V1.3 本地模型仍是增强体验，不是核心依赖。设备分级的目标是：

- 避免低端设备默认启用本地模型导致等待过长。
- 降低发热、耗电和内存风险。
- 保证模型失败时始终回退模板。
- 保证默认体验稳定。
- 保证 `TemplateTextRefiner` 仍是可靠兜底。
- 只在适合的设备上考虑 TestFlight 实验入口。

本地模型不能影响本地命理引擎、称骨计算、命格诗文、命理问答或知识库主流程。

## 2. 基于 benchmark 的初步判断

### iPhone 17 Pro Max

已知结果：

- 系统：iOS 26.2。
- 模型：`qwen2.5-0.5b-instruct-q4_k_m.gguf`。
- 正常样例 total 约 `0.96` 到 `1.15` 秒。
- 正常样例平均 total 约 `1.09` 秒。
- 高风险样例成功 fallback。

判断：

- 约 1 秒级。
- 适合进入 TestFlight 内测。
- 可作为高性能设备参考。
- 仍需补充 Instruments 的 Memory、CPU、Energy 和 Thermal 数据。

### iPhone 12 mini

已知结果：

- 系统：iOS 26.5。
- 模型：`qwen2.5-0.5b-instruct-q4_k_m.gguf`。
- 短安全提示 total 约 `4.57` 到 `5.10` 秒。
- 命理结果润色 total 约 `4.47` 秒。
- 今日建议润色 total 约 `2.50` 秒，并触发 fallback。
- 财运文本润色 total 约 `2.94` 秒。
- 高风险输入 total 约 `3.10` 秒，并触发 fallback。
- 出现过疑似提示词泄露：`<|im_end|>`。

判断：

- 可以运行，但体验一般。
- 不适合默认开启。
- 应归入“谨慎启用 / 默认关闭 / 用户手动实验”或“低端禁用”候选。
- A14 / 小内存设备在生产化中必须谨慎处理。

## 3. 设备分级建议

### Tier A：推荐内测启用

参考设备：

- 新款 Pro / Pro Max。
- A17 Pro / A18 / A19 或更新。

目标体验：

- 1 到 2 秒级短文本润色。
- 连续运行不崩溃。
- 不明显卡死 UI。
- 高风险输出能稳定 fallback。

策略：

- 允许 TestFlight 实验开关。
- 默认关闭。
- 用户手动开启。
- 必须支持随时回退 `TemplateTextRefiner`。
- 仍需要显示“本地模型仅用于文本润色”的说明。
- 首次开启必须展示实验说明和用户确认。

### Tier B：谨慎启用

参考设备：

- 较新的非 Pro 机型。
- 具体芯片和内存需要更多 benchmark 确认。

目标体验：

- 短文本润色可接受，但可能慢于 Tier A。
- 不应影响默认主流程。

策略：

- 默认关闭。
- 用户手动开启实验。
- 必须显示“可能较慢或耗电”的说明。
- 需要生成超时回退。
- 需要持续记录更多 benchmark。
- TestFlight 中可暂不显示，或显示为谨慎启用。

### Tier C：默认禁用

参考设备：

- iPhone 12 mini / A14 级别及更老设备。
- 小内存设备。
- benchmark 显示 4 到 5 秒级等待的设备。

策略：

- 保持 `TemplateTextRefiner`。
- 不推荐默认显示本地模型入口。
- TestFlight 开关默认不显示，或显示为不可用。
- 如果 TestFlight 需要测试，可通过明确实验名单或调试入口单独验证。
- 不进入默认用户路径。

## 4. 最低设备策略

初稿建议：

- V1.3 不设置硬性最低系统版本变更。
- 本地模型实验功能只对高性能设备开放。
- 生产前需要通过机型标识判断设备 tier。
- 若无法识别设备，默认禁用本地模型。
- 低电量模式、过热状态、内存压力异常时禁用或回退模板。
- App 的核心功能不得依赖本地模型。

最低策略原则：

- 识别不清：禁用。
- 设备偏老：禁用。
- 用户未开启：禁用。
- 模型异常：回退。
- 安全异常：回退。

## 5. 运行时禁用条件

即使设备属于 Tier A，也应在以下情况禁用或回退：

- 模型文件不存在。
- 模型加载失败。
- 生成超时。
- 输出安全检查失败。
- 输出为空。
- 输出过长。
- 输出包含提示词泄露或特殊 token。
- 低电量模式。
- 过热状态。
- 连续失败次数过多。
- 用户关闭开关。

运行时禁用不能影响原有结果页。用户仍应看到模板生成的稳定结果。

## 6. 超时策略

建议初稿：

| 设备等级 | 超时策略 |
| --- | --- |
| Tier A | 生成超过 `3` 秒回退 |
| Tier B | 生成超过 `5` 秒回退 |
| Tier C | 不启用 |

通用规则：

- 任何设备加载失败立即回退。
- 任何安全检查失败立即回退。
- 任何提示词泄露或特殊 token 输出立即回退。
- 任何设备连续失败次数过多时，本次会话禁用本地模型。

超时阈值需要在更多设备 benchmark 后调整。

## 7. 需要补充的 benchmark

建议补充设备：

- iPhone 15 / 15 Pro。
- iPhone 16 / 16 Pro。
- iPhone 13 / 14。
- iPad。
- 至少一台低内存设备。

建议记录指标：

- load time。
- generation time。
- total time。
- 连续 5 到 10 次稳定性。
- Xcode Instruments Memory。
- CPU。
- Energy。
- Thermal State。
- UI 是否卡顿。
- fallback 是否正常。
- 是否出现特殊 token 泄露。
- 是否出现重复文本。

## 8. 初步结论

- V1.3 不应全量默认启用本地模型。
- 推荐只对 Tier A 设备做 TestFlight 实验开关。
- iPhone 12 mini / A14 级别不建议默认启用。
- 默认体验继续使用 `TemplateTextRefiner`。
- 本地模型仅作为增强能力。
- 生产化前必须补充更多设备 benchmark 和 Instruments 数据。

## 9. TestFlight 开关关系

TestFlight 实验开关与设备 tier 的关系：

| 设备等级 | TestFlight 开关 |
| --- | --- |
| Tier A | 可显示，默认关闭，用户手动开启 |
| Tier B | 可显示但必须提示可能较慢或耗电，也可暂不显示 |
| Tier C | 默认不显示，或显示为不可用 |

开关只影响 `TextRefining` 润色实验，不影响命理计算、结果生成、历史记录原始结果或知识库内容。
