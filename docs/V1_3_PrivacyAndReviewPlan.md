# DestinyScope V1.3 Privacy And Review Plan

更新日期：2026-05-29

## 1. 隐私政策需要补充的内容

如果未来将本地模型能力开放到 TestFlight 或生产路径，隐私政策需要补充：

- 本地模型仅在设备端处理文本。
- 不上传出生信息。
- 不上传命理结果。
- 不上传模型输入。
- 不上传模型输出。
- 历史记录仍保存在本地设备。
- 模型失败时使用本地模板文本。
- 本地模型只做文本润色，不负责命理计算或结论生成。
- 如果未来支持模型下载，需要说明网络请求、下载文件、文件校验和本地存储。
- 如果未来支持日志或诊断，需要单独说明采集范围和用户控制方式。

当前阶段不更新 App 内隐私政策页面，也不更新 GitHub Pages 隐私页。V1.3 阶段 5 只准备草案。

## 2. App Store Review Notes 需要说明

如果本地模型进入 TestFlight 或生产路径，审核备注需要说明：

- 本地模型只做文本润色。
- 本地模型不生成命理计算结论。
- App 不提供医疗、法律、投资、婚恋、职业确定性建议。
- 结果仅供娱乐、自我探索和传统文化学习参考。
- 模型失败、安全检查失败或设备不满足条件时，会回退到模板文本。
- App 不收集用户数据。
- 出生信息和命理结果不上传。
- 若模型文件随 App 分发，需要说明模型来源和 license / notice。
- 若模型按需下载，需要说明下载目的和数据处理方式。

审核员测试步骤也需要单独说明：本地模型是否默认关闭、如何打开、如何验证回退、哪些设备可见。

## 3. App Store 元数据限制

元数据、截图、副标题、关键词和宣传文本必须避免：

- 精准算命。
- 精准预测未来。
- 改命。
- 化解。
- 转运承诺。
- 必然发财。
- 健康、寿命、疾病预测。
- 投资收益确定性。
- 婚姻确定性判断。
- AI 能准确预测人生。
- 大师在线或真人咨询暗示。

如果需要提到本地模型，推荐表述为：

- 本地文本润色。
- 设备端文本表达优化。
- 本地模型仅用于润色已生成的传统文化参考内容。

不建议在当前 App Store 元数据中宣传 AI / 本地模型，因为该能力尚未进入 Release。

## 4. License / Notice

当前 PoC 使用 Qwen2.5 相关 GGUF 文件。Qwen2.5 官方模型为 Apache 2.0，但生产化前仍需人工确认：

- 具体模型 repo URL。
- 具体 GGUF 文件 URL。
- GGUF 是否来自官方或可信量化仓库。
- license 类型。
- 是否允许商业使用。
- 是否允许再分发。
- 是否允许 App 内分发。
- 是否需要保留 license / notice。
- 是否需要在 App 内展示开源声明。
- 是否需要在 App Store 审核备注说明模型来源。
- 是否存在模型输出或 acceptable use 限制。

未完成 license / notice 审计前，不能随 App 分发模型文件，不能进入生产路径。

## 5. 阶段结论

V1.3 的隐私和审核工作应先产出草案，不直接修改 App 内隐私政策、GitHub Pages 隐私页或 App Store 元数据生产文案。

只有当本地模型确定进入 TestFlight 或生产路径时，才执行正式文案更新。
