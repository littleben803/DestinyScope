# DestinyScope V1.5 产品体验优化计划

## 1. V1.5 定位

V1.5 是产品体验优化与上架前产品打磨版本。

重点：

- 不新增高风险命理能力。
- 不扩大本地模型生产化范围。
- 不准备立即上架。
- 继续保持完全 Native、本地优先、隐私友好。
- 重点优化用户体验、内容结构、可读性、稳定性和可测试性。

V1.5 不改变当前核心决策：默认主流程继续由本地规则引擎和模板系统生成，本地模型实验保持受控、默认关闭、失败回退。

## 2. V1.5 核心目标

### P0 必须做

- 修复 source-control 风险：当前 `.gitignore` 的 `Models/` 规则可能误忽略 `DestinyScope/Domain/Models` 下新增 Swift 文件，例如 `OpenSourceLicenseItem.swift`。
- 做一次完整 UI / UX 审计。
- 优化结果页阅读体验。
- 优化知识库浏览体验。
- 优化历史记录体验。
- 完善本地模型实验功能的用户提示，但不扩大默认路径。
- 增强可访问性、深色模式、小屏适配。
- 增强 Debug / QA 检查清单。

### P1 可以做

- 知识库分类筛选。
- 知识库搜索。
- 历史记录详情页。
- 结果页折叠 / 展开。
- 首次打开说明页或轻量 onboarding。
- 分享文案草案，但不做图片分享。
- 截图素材规划更新。

### V1.5 明确不做

- 不做 App Store 正式发布。
- 不做 TestFlight 上传。
- 不做模型下载。
- 不做云端 AI。
- 不做 RAG。
- 不做流式 UI。
- 不做开放聊天。
- 不做付费订阅。
- 不做真人咨询。
- 不做改命、化解、避灾、转运类功能。
- 不做医疗、投资、婚恋确定性判断。
- 不把本地模型输出写入历史记录。
- 不让本地模型生成命理结论。

## 3. V1.5 与 V1.4 的关系

- V1.4 已证明受控 TestFlight 本地模型路径可设计。
- V1.5 暂不继续推进上传。
- V1.5 先打磨普通用户路径。
- 本地模型实验保持受控、默认关闭、失败回退。
- 普通结果页默认仍由规则引擎和模板系统生成。

V1.5 可以优化本地模型实验的说明文案和不可用状态提示，但不能把实验能力变成默认体验，也不能把模型结果写入历史或替代原始结果。

## 4. V1.5 成功标准

- Debug / Release 构建稳定。
- 默认主流程不依赖本地模型。
- 结果页更易读。
- 知识库更易浏览。
- 历史记录更可用。
- 小屏 / 深色模式 / 长文案可读性提升。
- 不引入新的隐私或审核风险。
- source-control 不再误忽略 Swift 源码。
- 文档和 checklist 可用于后续上架前复查。
