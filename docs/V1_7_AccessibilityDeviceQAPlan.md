# DestinyScope V1.7 Accessibility and Device QA Plan

## 1. V1.7 定位

V1.7 是多设备人工验收与无障碍修复版本。

重点：

- 不新增核心产品功能。
- 不考虑上架。
- 不上传 TestFlight。
- 不扩大本地模型范围。
- 重点检查并修复多设备、小屏、深色模式、VoiceOver、Dynamic Type、长文阅读问题。
- 保持默认主流程稳定，继续由本地规则引擎和模板系统生成结果。
- 保持 `makeDefaultRefiner()` 默认返回 `TemplateTextRefiner`。

## 2. 验收范围

页面范围：

- Onboarding。
- 首页 `HomeView`。
- 常用出生资料。
- 结果页 `DestinyResultView`。
- 复制与分享卡片。
- 本地润色预览卡片。
- 命理问答。
- 知识库列表。
- 知识库详情。
- 历史记录列表。
- 历史详情。
- 本地数据管理。
- 设置页。
- 关于页。
- 隐私政策。
- 免责声明。
- 开源许可。
- 本地模型实验设置。
- Debug 本地模型页面，仅 Debug 检查。

## 3. 设备范围

必须覆盖：

- iPhone SE 小屏。
- iPhone 12 mini / mini 类小屏。
- 标准 iPhone。
- iPhone 17 Pro Max 或 Pro Max 类大屏。
- iPad。
- Simulator。

记录原则：

- 如暂无真机，状态标记为“待人工验证”。
- 不伪造通过结果。
- 每条人工记录应写明设备、系统、App build、测试范围和问题截图 / 录屏路径。

## 4. Accessibility 检查范围

检查项：

- VoiceOver 焦点顺序。
- 关键按钮 `accessibilityLabel`。
- 关键按钮 `accessibilityHint`。
- 删除 / 清空 / 导入 / 生成 / 分享 / 复制按钮可理解。
- 空状态可理解。
- 错误状态可理解。
- Sheet / `confirmationDialog` 文案可理解。
- TabView 读屏顺序。
- Legal 长文读屏顺序。
- 开源许可长 URL 不影响 VoiceOver 理解。

重点判断：

- 用户是否能不用视觉完成首页查询、知识库阅读、历史删除、数据清理等核心动作。
- 危险操作是否在 VoiceOver 下明确说明后果。
- 本地模型实验入口是否被朗读成受控实验，而不是默认命理能力。

## 5. Dynamic Type 检查范围

检查尺寸：

- Default。
- Large。
- Extra Large。
- Accessibility Large。

重点页面：

- Onboarding。
- 首页输入区。
- 结果页卡片。
- 知识库列表和详情。
- 历史列表和详情。
- 设置 / Legal 长文。
- 本地数据管理。

目标：

- 不截断关键按钮。
- 不因固定高度导致内容不可见。
- tags / chips / badge 可以换行或滚动。
- 长文可滚动阅读。
- 危险操作确认文案在大字号下仍完整可见。

## 6. 深色模式检查范围

检查项：

- 背景与卡片层级。
- 主文字 / 次文字对比。
- 朱砂红 / 暗金色对比。
- 标签 / badge / chip 对比。
- 错误提示可读。
- Legal 长文可读。
- 按钮可见。
- 本地模型实验警示说明可读。

目标：

- 任何关键操作不应依赖低对比颜色。
- 长文本阅读不应出现疲劳或低对比段落。
- 禁用、危险、确认状态应在深色模式下仍能区分。

## 7. 小屏检查范围

重点设备：

- iPhone SE。
- iPhone 12 mini。

检查项：

- 首页是否过长但可滚动。
- DatePicker / Picker 是否可操作。
- 查询按钮是否可见。
- 常用资料是否不挤压。
- 结果页权重明细是否不溢出。
- 知识库分类 chip 是否可横向滚动。
- 历史记录卡片是否不拥挤。
- Legal 页面是否不被固定高度截断。
- 本地数据管理危险按钮是否可见且不误触。

## 8. 本地模型实验路径检查

必须确认：

- Release 不展示实验入口。
- 实验开关默认关闭。
- 未接受说明不能开启。
- Tier C 不可用。
- 模型不存在时不可用。
- 生成需要用户手动点击。
- 原始结果始终保留。
- 失败回退。
- 不写历史。
- `makeDefaultRefiner()` 仍返回 `TemplateTextRefiner`。

V1.7 不扩大本地模型能力，不新增模型下载，不接 RAG，不做开放聊天。

## 9. 风险分级

P0：

- 阻断主流程。
- 隐私误导。
- 误触危险删除。
- 读屏无法使用。
- Release 暴露实验入口。
- `makeDefaultRefiner()` 默认路径被改变。
- 新增网络、权限、支付、追踪或模型下载。

P1：

- 小屏拥挤。
- Dynamic Type 截断。
- 深色模式对比不足。
- 长文阅读困难。
- 重要按钮 label / hint 不完整。
- 本地模型实验说明不够克制。

P2：

- 文案可进一步简化。
- 视觉细节。
- 轻微间距问题。
- 非阻断性排版细节。

## 10. 验收输出

V1.7 阶段 2 应输出：

- 人工验收记录汇总。
- 按 P0 / P1 / P2 分类的问题列表。
- 可复现步骤。
- 截图 / 录屏路径。
- 建议修复阶段。

V1.7 后续修复必须保持小步修改，每次修复后运行 Debug / Release 构建，并确认默认主流程和本地模型边界未改变。
