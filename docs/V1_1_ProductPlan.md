# DestinyScope V1.1 产品规划

更新日期：2026-05-27

## 1. V1.1 定位

DestinyScope V1.1 是 V1.0 的离线体验增强版。它继续沿用 V1.0 的核心边界：

- 仍然完全 Native。
- 仍然无服务端。
- 仍然无在线 AI。
- 仍然不接真实本地 LLM。
- 出生信息仍然只在设备端处理。

V1.1 的重点不是立刻接入本地小模型，而是把现有离线体验做厚：提升结果页信息密度、增强互动性、扩充知识库、建立本地历史记录，并为后续 0.5B 到 1.5B 本地小模型预留结构化输入和润色接口。

产品表达上，V1.1 仍应避免被包装成单纯“算命 App”。它应继续强调东方命理文化、自我探索、本地隐私保护和传统知识学习。结果内容需要保持参考性，避免绝对预测、恐吓式内容和现实决策承诺。

## 2. V1.1 核心目标

- 增强结果页报告结构：在现有称骨结果、诗文、权重明细和五类解读基础上，新增更清晰的报告层级。
- 新增 `LifeWeightInsight`：输出命格标签、优势倾向、关注点、行动建议。
- 新增命理问答入口：用户可以围绕结果提出预设问题，但 V1.1 只使用本地模板回答。
- 扩充知识库：从当前 5 篇扩充到至少 20 篇，提高文化学习和内容厚度。
- 建立知识来源和未来 RAG 资料结构：记录资料来源、整理原始笔记和文章结构。
- 新增本地历史记录：让用户可以在设备端查看过去生成的结果。
- 预留本地小模型润色接口：只定义协议和 Mock/Template 实现，不接真实模型。

## 3. V1.1 功能范围

### P0 必须做

- `LifeWeightInsight`：命格标签、优势倾向、关注点、行动建议。
- 结果页展示更完整的报告结构。
- 命理问答入口。
- 预设问题回答：事业、财运、性格、关系、今日建议。
- 知识库扩充到至少 20 篇。
- 知识来源整理文档。
- 本地历史记录。
- 本地小模型润色接口协议和 Mock/Template 实现。

### P1 可以做

- 知识库分类筛选。
- 知识库搜索。
- 结果分享文案。
- 历史记录删除/重命名。
- 今日建议卡片。

P1 只在 P0 稳定后进入，不应影响 V1.1 主链路。

### V1.1 明确不做

- 真实本地 LLM 推理。
- 模型下载。
- 服务端。
- 登录。
- 在线 AI。
- 付费订阅。
- 真人咨询。
- 投资、健康、寿命、婚姻确定性判断。
- 恐吓式文案和化解类功能。

## 4. V1.1 与后续第二阶段关系

V1.1 先把结构化输入、模板回答、知识库和问答入口做好。后续第二阶段如果引入 0.5B 到 1.5B 本地小模型，小模型只应负责润色和自然语言表达，不负责命理计算和结论生成。

职责边界建议：

- `LifeWeightEngine` 继续负责确定性称骨计算。
- `LifeWeightInsight` 由本地规则和模板生成。
- 命理问答答案由本地模板基于 `LifeWeightResult`、`LifeWeightInsight` 和预设问题生成。
- 本地小模型只接收已生成的结构化结论和安全边界，做语言润色。
- 小模型不得新增未由规则生成的结论，不得编造四柱、五行、十神、大运、流年等当前数据结构没有的信息。

## 5. 文案与安全边界

V1.1 新增内容必须继续遵守：

- 使用“倾向、参考、建议、可以关注、自我探索”等表达。
- 不使用“必然、注定、一定发财、必有灾祸、必离婚、必生病”等绝对化或恐吓式表达。
- 不生成健康、寿命、疾病预测。
- 不生成投资确定性建议。
- 不生成婚姻绝对判断。
- 不提供化解、改命、避灾类服务。
- 每个结果页或问答页都保留“仅供娱乐、自我探索和传统文化学习参考”的提示。

## 6. V1.1 验收标准

- 用户可以从结果页看到更完整的报告结构。
- 用户可以看到命格标签、优势倾向、关注点、行动建议。
- 用户可以进入命理问答入口，并选择事业、财运、性格、关系、今日建议等预设问题。
- 所有问答答案由本地模板生成，不请求网络，不接 AI SDK。
- 知识库至少 20 篇，并保留 category、tags、source、version。
- 有知识来源整理文档。
- 用户可以在本地查看历史记录。
- 已有隐私政策和免责声明仍然准确，不需要因为 V1.1 增加服务端或在线 AI 说明。
- App Store 元数据继续避免真实 AI、精准预测和现实决策承诺。
