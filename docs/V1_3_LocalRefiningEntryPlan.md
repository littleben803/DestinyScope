# DestinyScope V1.3 Local Refining Entry Plan

更新日期：2026-05-29

## 1. 阶段定位

本阶段只设计本地模型润色入口，不实现 Swift 代码。

边界：

- 本地模型润色不进入默认主流程。
- 不自动替换结果页原始解读。
- 不改变称骨计算。
- 不改变命格标题、诗文、权重、命格洞察或命理问答结论。
- 不替代 `LifeWeightEngine`、`TemplateFortuneInterpreter`、`LifeWeightInsightGenerator` 或 `TemplateFortuneQuestionAnswerer`。
- 只作为可选“润色预览”。

本地模型只负责文本表达润色，不能生成新的命理结论。

## 2. 候选入口方案

### 方案 A：结果页底部增加“本地润色预览”卡片

特点：

- 只在 TestFlight 实验开关开启且设备支持时展示。
- 用户点击“生成润色版”。
- 展示润色后的总结文本。
- 原始结果仍保留。
- 不覆盖任何原始模板内容。

优点：

- 入口清晰。
- 对默认主流程影响最小。
- 只触发一次生成，性能风险可控。
- 便于比较原始文本和润色预览。

结论：推荐作为 V1.3 首选。

### 方案 B：结果页每个区块旁增加“润色”按钮

特点：

- 粒度更细。
- 每个区块可单独润色。

风险：

- UI 复杂。
- 调用次数多。
- 性能风险高。
- 容易让用户误以为模型参与了每个命理结论。

结论：暂不推荐。

### 方案 C：命理问答回答后提供“润色这段回答”

特点：

- 与问答交互自然衔接。
- 适合后续增强问答体验。

风险：

- 容易和问答结论生成混淆。
- 需要额外设计问答结果的保存、刷新和回退。

结论：适合后续，当前先不做。

### 方案 D：设置页提供独立“本地润色实验”页面

特点：

- 安全、隔离。
- 与主流程无耦合。
- 适合 Debug / TestFlight 验证。

缺点：

- 产品价值弱。
- 用户不容易理解它和结果页内容的关系。

结论：可作为 Debug / TestFlight 验证补充，不作为 V1.3 首选产品入口。

## 3. 推荐方案

V1.3 首选方案：

- 方案 A：结果页底部“本地润色预览”卡片。
- 默认不展示。
- 用户必须手动点击生成。
- 原始结果始终保留。
- 润色结果只作为当前页面临时预览。

展示条件：

- TestFlight 实验开关已开启。
- 设备属于 Tier A 或允许的 Tier B。
- 模型文件存在。
- 当前不处于低电量状态。
- 当前不处于过热状态。
- 当前没有连续失败过多。

失败策略：

- 生成失败回退模板。
- 显示温和错误。
- 不覆盖原始文本。

## 4. 展示内容

卡片标题建议：

- 本地润色预览。
- 本地文本润色实验。

说明文案建议：

```text
使用设备端本地模型对已有模板文本做表达润色，不生成新的命理结论。
结果仅供娱乐、自我探索和传统文化学习参考。
如生成失败，将自动保留原始模板文本。
```

按钮建议：

- 生成润色版。
- 重新生成。
- 使用原始文本。

状态文案建议：

- 正在生成润色预览...
- 本地润色暂不可用，已保留原始模板文本。
- 模型输出未通过安全检查，已回退到原始文本。

## 5. 输入范围

首期只允许输入：

- `FortuneInterpretation.summary`。
- `LifeWeightInsight.actionSuggestion`。
- 已有安全提示。

不允许输入：

- 用户完整出生信息。
- 原始生日详情。
- 历史记录列表。
- 知识库全文。
- 任何用户自由输入。
- 未经过结构化处理的数据。

输入应尽量短，避免把完整结果页内容全部交给模型。

## 6. 输出范围

输出只作为：

- 当前页面临时预览。
- 可选的润色后文案展示。

输出不应：

- 默认写入历史记录。
- 覆盖原始结果。
- 作为新的命理结论。
- 参与后续问答推理。
- 进入知识库或训练数据。

如果未来要保存润色结果，必须单独设计：

- 保存字段。
- 用户确认。
- 隐私说明。
- 删除机制。
- 是否同步或导出。

## 7. 回退与错误文案

回退场景：

- 模型不存在。
- 设备不支持。
- 低电量。
- 过热。
- 加载失败。
- 生成超时。
- 输出为空。
- 安全检查失败。
- 用户取消。
- 连续失败次数过多。

用户文案：

- 本地润色暂不可用，已保留原始模板文本。
- 当前设备可能较慢，建议继续使用原始解读。
- 模型输出未通过安全检查，已回退到原始文本。

不展示：

- 模型路径。
- framework 错误栈。
- llama.cpp 内部错误。
- 调试 token。

## 8. 体验约束

- 不应阻塞整个结果页。
- 显示 loading。
- 可取消。
- 超时自动回退。
- 不自动连续生成。
- 不做流式输出。
- 不做聊天 UI。
- 不做 RAG。
- 不做模型下载。
- 不默认保存输出。

推荐交互：

1. 用户进入结果页。
2. 如果开关、设备和模型状态满足条件，底部显示“本地润色预览”卡片。
3. 用户点击“生成润色版”。
4. 卡片内显示 loading。
5. 成功则显示润色文本。
6. 失败则显示温和错误，并保留原始模板文本。

## 9. 安全与合规

必须继续遵守：

- 模型只能润色。
- 不能新增命理结论。
- 不能编造四柱、五行、十神、大运、流年。
- 不能做医疗、法律、投资、婚恋、职业确定性建议。
- 不能输出精准预测、改命、化解、避灾、必然发财等内容。
- 安全检查失败必须回退。
- 结果仍仅供娱乐、自我探索和传统文化学习参考。

如果该入口进入 TestFlight，需要配套：

- 隐私说明：本地模型仅在设备端运行，不上传出生信息、命理结果、模型输入或模型输出。
- 审核备注：说明本地润色只做文本表达优化，不生成新的命理结论。
- 元数据限制：不得宣传为 AI 算命、精准预测、改命、化解或转运。
- 用户说明：首次开启前告知实验性质、可能较慢或失败、失败会回退模板。
- 免责声明：继续说明结果仅供娱乐、自我探索和传统文化学习参考。

## 10. TestFlight 验收标准

- 实验开关关闭时不展示入口。
- Tier C 不展示入口。
- 模型不存在时入口不可用或点击后回退。
- 点击生成后原始结果仍可见。
- 生成结果不覆盖原始文本。
- 安全失败会回退。
- 超时会回退。
- App 重启后不会把润色结果误认为原始结论。
- Release 默认不展示。
- 不保存用户完整输入和模型输出。
- 不上传任何润色相关日志。

## 11. 阶段结论

V1.3 推荐采用结果页底部“本地润色预览”卡片作为 TestFlight 内测入口。该入口必须由用户手动触发，只生成当前页面临时预览，不覆盖原始结果，不写入历史记录，不参与后续命理问答推理。
