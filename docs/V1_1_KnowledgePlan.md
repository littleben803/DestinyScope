# DestinyScope V1.1 知识库规划

更新日期：2026-05-27

## 1. 知识库扩充目标

V1.1 目标是将当前本地知识库从 5 篇扩充到至少 20 篇。知识库同时服务两个方向：

- App 展示：为用户提供传统命理、农历、时辰、生肖、五行、八卦等入门内容。
- 未来 RAG 准备：为后续本地小模型润色和问答提供结构化资料基础。

内容原则：

- 原创改写。
- 通俗简短。
- 保持参考性和文化学习属性。
- 不复制来源不明的大段网络内容。
- 不加入恐吓式、化解式、营销式玄学话术。
- 不输出健康、寿命、疾病、投资收益、婚姻结果的确定性判断。

## 3. 资料来源策略

优先使用：

- 公开、可引用、公版或机构类资料。
- 古籍原文或公版整理版本。
- 图书馆、大学、博物馆、研究机构公开资料。
- 公开 PDF 或开放数据库。
- 权威出版社或学术资料中的概念性描述，注意只做摘要和改写。

谨慎使用：

- 百科类条目，只用于概念核对，不复制大段正文。
- 个人博客，只作为线索，不作为主要来源。
- 视频口播内容，只作为主题参考，不直接转写。

避免使用：

- 来源不明的大段网络文章。
- 营销号、卖课页、付费咨询页。
- 恐吓式玄学页面。
- 化解、改命、开运商品导购内容。
- 明确带有版权限制且未授权的现代书籍大段摘录。

## 4. 资料结构建议

建议新增资料目录：

```text
docs/knowledge_sources/sources.md
docs/knowledge_sources/raw_notes.md
```

现有 App 资源：

```text
DestinyScope/Resources/Knowledge/knowledge_articles.json
```

后续预留资源：

```text
DestinyScope/Resources/Knowledge/rag_chunks.json
```

文件职责：

- `docs/knowledge_sources/sources.md`：记录每篇文章参考过的来源、URL、书名、机构、访问日期和使用方式。
- `docs/knowledge_sources/raw_notes.md`：记录人工整理后的要点，不直接复制原文。
- `knowledge_articles.json`：App V1.1 展示用文章。
- `rag_chunks.json`：后续 RAG 预留，不在 V1.1 接入。

## 5. KnowledgeArticle 字段建议

继续沿用 V1 的文章结构：

```json
{
  "id": "lunar_calendar_basics",
  "category": "农历时辰",
  "title": "农历基础",
  "summary": "用通俗方式解释农历和公历的差异。",
  "body": "300 到 600 字原创改写正文。",
  "tags": ["农历", "传统文化", "入门"],
  "source": "DestinyScope 原创整理；参考公开资料",
  "version": "1.1"
}
```

约束：

- `id` 使用稳定英文 snake_case。
- `category` 用于列表展示和后续筛选。
- `summary` 控制在 30 到 80 字。
- `body` 建议 300 到 600 字。
- `tags` 保持 2 到 5 个。
- `source` 不写成模糊的“网络资料”，应尽量标明来源类别。
- `version` 对 V1.1 新增内容可使用 `1.1`。

## 6. RAG 预留策略

V1.1 不接向量数据库，不接本地 LLM，不做真实 RAG。但文章结构需要为后续拆分做准备：

- 每篇文章控制在 300 到 600 字，天然适合后续拆成 1 到 2 个 chunk。
- 每篇文章保留 `category`、`tags`、`source`、`version`。
- 避免正文混入多个无关主题。
- 文章应有清晰主旨，便于后续语义检索。
- 后续 `rag_chunks.json` 可以从 `knowledge_articles.json` 派生。

未来 `rag_chunks.json` 结构可参考：

```json
{
  "id": "lunar_calendar_basics_chunk_001",
  "articleId": "lunar_calendar_basics",
  "category": "农历时辰",
  "text": "chunk text",
  "tags": ["农历", "传统文化"],
  "source": "same as article",
  "version": "1.1"
}
```

## 7. 合规边界

知识库文章应保持文化解释和学习属性：

- 可以解释历史背景、术语来源、象征意义和现代使用边界。
- 可以说明“传统命理常如何理解某概念”。
- 不应给出具体个人命运断言。
- 不应引导用户购买化解服务或咨询服务。
- 不应输出医疗、投资、婚恋、法律、职业等现实决策建议。

每篇涉及命理结果使用方式的文章，都应包含类似提示：

```text
相关内容仅供娱乐、自我探索和传统文化学习参考，不应作为现实重大决策依据。
```
