# DestinyScope V1.5 UX 审计计划

## 1. 审计范围

页面：

- 首页
- 结果页
- 本地润色预览卡片
- 命理问答
- 知识库列表
- 知识库详情
- 历史记录
- 设置页
- 关于页
- 隐私政策
- 免责声明
- 开源许可

## 2. 审计维度

- 信息层级。
- 文案清晰度。
- 是否有高风险表述。
- 卡片层级是否过多。
- 长文本是否可读。
- 小屏 iPhone 是否截断。
- iPad 是否留白过大。
- 深色模式是否可读。
- Dynamic Type 是否可用。
- VoiceOver 标签是否需要补充。
- 空状态和错误状态是否友好。
- 本地模型实验入口是否会误导用户。
- 历史记录是否清楚说明仅本地保存。

## 3. 审计输出格式

每个页面记录：

```markdown
## 页面名称

当前问题：

- ...

优先级：

- P0 / P1 / P2

建议改动：

- ...

风险：

- ...

是否需要代码变更：

- 是 / 否

是否需要文案变更：

- 是 / 否

是否影响隐私 / 审核：

- 是 / 否
```

## 4. 优先级定义

P0：

- 影响主流程理解、导致误导、可能触发审核或隐私风险、明显小屏截断、深色模式不可读。

P1：

- 影响阅读效率、信息层级、局部交互清晰度，但不阻断主流程。

P2：

- 视觉细节、文案润色、轻量排版优化。

## 5. 高风险表述检查

审计时必须确认页面和截图文案不出现以下营销式表达：

- 精准预测
- 改命
- 化解
- 避灾
- 保证转运
- 必然发财
- 大师在线
- 健康寿命预测
- 投资收益确定
- 婚姻确定性判断

允许表达：

- 传统文化学习
- 自我探索
- 参考性解读
- 本地处理
- 本地文本润色
- 不生成新的命理结论
- 失败回退模板
